# Опис проєкту

Контрольна робота №1 з дисципліни Бази даних (3 семестр)

Умова (коротко):

- Спроєктувати схему бази даних з 5-6 таблиць
- Написати 8 OLTP-запитів (по 2 INSERT, UPDATE, DELETE, SELECT)
- Написати 4 OLAP-запити (аналітичні запити)

## Необхідні сутності

- Ресторани (назва, кухня, адреса, рейтинг)
- Клієнти (адреса доставки, платіжна інформація)
- Кур'єри (транспортний засіб, статус доступності, район)
- Замовлення (час замовлення, час доставки, статус, загальна сума)
- Позиції замовлення (страви з ресторану з кількістю)

## Файлова структура (існуючі файли)

- `docker-compose.yml` — конфігурація для запуску сервісів
- `src/create-tables.sql` — тут розміщуйте DDL (схема бази даних)
- `src/data_manipulation.sql` — тут розміщуйте OLTP-запити (INSERT/UPDATE/DELETE/SELECT)
- `src/analitical.sql` — тут розміщуйте OLAP/аналітичні запити

## Вимоги — детально

1) Схема бази даних

- Створіть 5–6 таблиць, що покривають перелічені сутності.
- Врахуйте первинні та зовнішні ключі, індекси для часто фільтрованих полів (наприклад, `restaurant_id`, `order_time`, `courier_id`).

2) OLTP запити (8 штук)

- 2x INSERT (додати запис до будь-яких таблиць)
- 2x UPDATE (змінити записи; наприклад, оновити статус доставки або рейтинг ресторану)
- 2x DELETE (видалити записи; наприклад, старі тестові дані)
- 2x SELECT (прості запити):
  - знайти всіх доступних кур'єрів у конкретному районі
  - вивести список усіх замовлень для ресторану за сьогодні

3) OLAP запити (4 штук, аналітика)

- Обчислити загальний дохід за рестораном за місяць
- Знайти топ-10 ресторанів з найвищим рейтингом та найбільшою кількістю замовлень
- Обчислити середній час доставки за кур'єром
- Визначити кур'єрів з найбільшою кількістю скасованих доставок (використати CTE)

## Прийнятні умови і критерії оцінки

- Схема створена коректно: таблиці, PK, FK, необхідні індекси
- OLTP: 8 SQL-запитів (2 INSERT, 2 UPDATE, 2 DELETE, 2 SELECT) коректно працюють на тестовій БД
- OLAP: 4 аналітичні запити повертають очікувані результати для тестових даних
- Коментарі/пояснення перед кожним запитом
